Clarity-first UI language

Build interfaces with Kattour — not scattered layers

Kattour is an experimental language platform for building modern user interfaces. One language, one file, one clear mental model.

Why Kattour

A cleaner way to think about UI

Every major framework adds complexity. Kattour removes it.

Core

Declarative by design

Describe what you want, not how to build it. Kattour reads like a design spec — structured, intention-revealing, human-first.

Architecture

Real compiler pipeline

A full tokenizer, parser, AST generator, diagnostics engine, and HTML renderer. Not templates — actual compilation.

Reactive

State built in

Platform

Reusable

Component model

Define reusable components with parameters. Compose UIs like building blocks — readable, predictable, maintainable.

Future

Every surface

Designed with web, mobile, desktop, and SSR in mind. One language for every surface you build on.

Syntax

Clean. Readable. Intentional.

Write once, render anywhere

Kattour syntax is designed around clarity. Each keyword maps directly to a UI concept — no ceremony, no boilerplate.

Screens, columns, rows, cards, buttons, and inputs are all built in. State is declared with one word. Routes are first-class.

MinimalExpressiveTyped
page Dashboard

theme {
  primary "#cb0606"
  radius 16
}

state count = 0
state user = "Elias"
computed greeting = "Hello, " + $user

route "/" {
  screen {
    column {
      title "$greeting"

      card {
        text "Sessions: $count"
        row {
          button "Add" { click count++ }
          button "Sub" { click count-- }
        }
      }

      link "Sign out" { to "/logout" }
    }
  }
}

Quick Start

Up and running in minutes

1. Install

Install the Kattour CLI globally.

npm install -g kattour

2. Create

Scaffold a new Kattour project.

kattour create my-app

3. Dev

Start the dev server with live reload.

cd my-app
kattour dev

Built with Kattour

This website is Kattour

Dogfooding from day one

Every page of this website — including the one you are reading now — is defined in a single app.kat file and rendered by the Kattour compiler.

The nav, sections, feature cards, code blocks, and footer are all Kattour language constructs. No HTML written by hand.

// This website in Kattour
page KattourWebsite

component NavBar {
  nav {
    title "Kattour"
    link "Docs" { to "/docs" }
    link "Examples" { to "/examples" }
  }
}

route "/" {
  screen {
    NavBar
    hero {
      eyebrow "Clarity-first"
      title "Build with Kattour"
      button "Get Started"
    }
    SiteFooter
  }
}
