# Website Rebuild Plan

## Goal
Preserve the visual identity of Kattour while replacing the current static structure with a maintainable application architecture.

## Website responsibilities
- explain what Kattour is
- provide docs and examples
- expose the roadmap
- host the playground
- communicate product maturity clearly

## Planned sections
- homepage
- docs
- examples
- roadmap
- playground
- downloads or install guide later

## Technical direction
- reusable components
- better navigation structure
- code examples tied to the actual language direction
- cleaner separation between marketing and product documentation

## Critical fix
The playground should stop relying on generic YAML parsing and move toward a true Kattour parsing flow.
