Getting Started

Install the CLI, create an app, and start the local dev renderer.

npm install -g kattour
kattour create my-app
cd my-app
kattour dev app.kat
Language

Core Syntax

Kattour uses explicit blocks. Every file starts with a page, followed by state and routes.

page Home

route "/" {
  column {
    title "Hello Kattour"
    text "Build readable UIs."
    button "Start"
  }
}
Layout

Layout Blocks

Use section, column, grid, card and stack blocks to build structured product interfaces.

section {
  title "Features"
  grid {
    card "Fast"
    card "Readable"
    card "Typed"
  }
}

Layout mindset

Describe intent, not DOM noise. The renderer decides the final optimized structure.

Runtime

State

State defines values used by UI blocks. Computed values are derived from state.

state count = 0
computed label = "Count: " + $count

button "$label" {
  click count++
}
App

Routing

Routes map URLs to UI blocks and expose dynamic route parameters.

route "/user/:name" {
  section {
    title "$route.params.name"
    text "User profile"
  }
}
Reuse

Components

Components are reusable blocks. They will become the foundation of the Kattour UI package system.

component ProductCard(name, price) {
  card "$name" {
    text "$price"
    button "Buy now"
  }
}
Interaction

Events

Events live inside interactive blocks and update state or trigger runtime effects.

button "Add to cart" {
  click cart.count++
}
